# CUMCM 优秀论文风格规则（截图抽样版）

本文件根据 2023、2024 年多篇 C 题优秀论文页面截图归纳，用于约束 AI 生成国赛论文模板和示例稿。它只总结结构与版式，不复用优秀论文文字、图表、代码或结果。

## 1. 标准章节顺序

推荐顺序：

1. 摘要页：标题、摘要、关键词；不放目录。
2. 一、问题重述：含“问题背景”和“问题要求/问题提出”。
3. 二、问题分析：按小问分节，不单独设置“总体思路”小节。
4. 三、模型假设：编号列表，每条按“假设：解释”写。
5. 四、符号说明：三线表，列为“符号、说明、单位”。
6. 五、数据预处理：数据来源、缺失异常、参数构造；不要写成“数据读取与预处理”。
7. 六、模型建立与求解：按问题一、问题二、问题三展开。
8. 七、模型检验：误差、敏感性、鲁棒性、可行性或对比验证。
9. 八、模型评价与推广：优点、局限、推广场景。
10. 参考文献。
11. 附录：支撑材料文件列表、源程序代码说明、AI 工具使用详情说明。

## 2. 标题命名约束

- 一级标题用中文数字加顿号：`一、问题重述`、`二、问题分析`。
- 二级标题用阿拉伯编号：`1.1 问题背景`、`2.1 问题一的分析`。
- 数据章节统一叫 `数据预处理`。
- 不在正文中设置 `编译测试结果`、`LaTeX 测试说明`、`算法实现说明` 这类非论文型章节；代码运行说明应放到附录或支撑材料 README。
- 不单列空泛 `结论` 章，除非题目或论文逻辑确实需要；通常在摘要、结果分析和模型评价中收束结论。
- 不设置 `2.4 总体思路`、`总体模型思路`、`整体思路` 等教学式小节；若需要说明路线，写入各小问分析自然段或使用“模型流程图”。

## 3. 模型假设写法

推荐：

```tex
\begin{enumerate}
  \item \textbf{假设 1：} 假设内容。解释：说明该假设对应的公式、约束或数据处理理由。
  \item \textbf{假设 2：} 假设内容。解释：说明影响和后续检验方法。
\end{enumerate}
```

避免只写短句列表，也避免把题目核心难点直接假设掉。

## 4. 摘要与关键词写法

摘要按小问组织：`针对问题一...`、`针对问题二...`、`针对问题三...`。每问要有方法和结果；没有运行结果时必须使用明确占位，不能编造数值。

关键词必须优先使用数学建模教材和竞赛论文中常见的模型、方法、算法、评价或检验术语，例如 `混合整数规划`、`目标规划`、`蒙特卡洛模拟`、`风险决策`、`敏感性分析`。避免把题目对象或行业场景直接作为关键词，例如 `农作物种植策略`、`乡村农业`、`蔬菜销售`。

## 5. 图表写法

- 如需展示模型路线，推荐使用“模型流程图”，但不要因此新增“总体思路”小节；流程图必须包含数据输入、模型模块、算法求解和结果输出。
- 符号表用三线表，字段含单位。
- 数据预处理节可以放数据字典、缺失值统计、描述统计或参数估计表。
- 图表标题必须具体，正文必须引用。


## 6. 参考文献与交叉引用

- 国赛论文模板默认使用 `ref.bib` 管理参考文献，正文用 `\cite{...}` 引用。
- 图表公式引用与文献引用必须分开：`\ref` / `\eqref` 只用于图、表、公式、章节；`\cite` 只用于 `ref.bib` 中的文献、数据源、软件或 AI 工具条目。
- 不为占位文字强行添加 `表~\ref{...}`、`图~\ref{...}`。只有当图表已经存在且正文确实需要编号引用时再添加。
- 参考文献前不使用 `\nocite{placeholder-ref}`；最终参考文献列表应能追踪到正文中的引用位置。
- 交付前运行 `python scripts/check_latex_refs.py main.tex --bib ref.bib`，确保没有未定义标签、重复标签或未定义文献键。
- CUMCM 模板应保持 `hypertexnames=false`，避免附录或浮动体编号变化导致 PDF 内部链接锚点重复，从而出现跳转到错误位置的问题。
